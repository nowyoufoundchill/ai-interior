## Fixes made this cycle

See BUILD_PLAN.md / SESSION_LOG.md for the full list; summary:

- Render route treated a mock `null` image as a live OpenAI failure when real keys happened to be configured (our seed:test fallback case) — spurious 500 in AI_MODE=mock.
- Photo add/delete never marked the current diagnosis stale (§4 row 1 gap).
- Editing a locked concept was silently allowed instead of rejected (§4 row 4).
- Unlocking left downstream products/renders non-stale until the next re-lock instead of immediately (§4 row 3).
- `ai_runs` and every artifact table written by a live route never carried `test_run_id`, so automated cycles would leak permanent, undetectable residue into production despite the residue check reporting clean.
- Tap targets across nav/tabs/buttons were under the 44px minimum (PRD §8).
- Debug-run count/link was visible in the primary room workspace, contradicting "hidden route... not an admin panel" (PRD §3/§8).
- Primary nav labeled "Dashboard" — dashboard-y language contradicting the design-studio register requirement.
- Mock concept fixtures (`styleDirector()`) used one identical hardcoded palette and templated sentence for all 3 concepts, failing the explicit "three concepts that feel the same" ban even in test/mock mode.
- Suite script bugs (not app bugs): `locator.count()` has no auto-retry; Next.js RSC prefetch aborts were misflagged as failed requests; tab-switch screenshots raced React's re-render and a CSS transition.
